🔗 OpenAI proxy (VPN)

How to fill in the fields

Manual entry (if you are not using quick paste):

  • Type — SOCKS5 / SOCKS5h for a SOCKS proxy (often port 1080); HTTP or HTTPS if your provider gave you an HTTP proxy (CONNECT).
  • Host — IP address or hostname of the proxy server.
  • Port — proxy port on that host.
  • Username / Password — credentials if the proxy requires auth; otherwise leave blank.
  • Full URL — a single line that overrides host/port/type below, e.g. socks5://user:pass@host:1080.

String format 1 (SOCKS5, test data only — not a real service):

socks5server=198.51.100.10&port=1080&user=vpn_user&pass=vpn_pass

Mapping: socks5server (or server) → Host; port → Port; user → Username; pass → Password. After pasting, click Parse into fields; type will switch to SOCKS5.

String format 2 (often an HTTP proxy, test example):

proxyserver=203.0.113.50&port=8443&secret=dGVzdC1zZWNyZXQtdG9rZW4tZXhhbXBsZQ

Mapping: proxyserver → Host; port → Port; secret is copied to Password (add Username yourself if required). Choose HTTP or HTTPS per your provider. A long secret from clients like VLESS is not a SOCKS/HTTP proxy by itself — run a local proxy (e.g. 127.0.0.1:1080) and enter that endpoint here.

Ping (ICMP) to the proxy host does not prove SOCKS works for HTTPS — use Test egress IP. If OpenAI still returns unsupported_country_region_territory, Via proxy must show an IP in a supported country (often US/EU) and differ from Direct. The app uses socks5h (remote DNS) when you choose SOCKS5. As a last resort, set API base URL to an OpenAI-compatible provider (e.g. OpenRouter) in an allowed region.
